# 🚀 GitHub Issue Tracker

A modern, responsive web application designed to track and manage project issues efficiently. This project focuses on API integration, dynamic UI updates, and core JavaScript principles.

## 🔗 Live Demo
[View Live Project](https://github-issue-trackers.vercel.app/)

## ✨ Key Features
- **Secure Authentication:** Protected login page with demo credential validation.
- **Dynamic Dashboard:** Real-time data fetching and rendering using the Fetch API.
- **Advanced Filtering:** Switch between 'All', 'Open', and 'Closed' issues seamlessly.
- **Live Search:** Search for specific issues using the integrated search bar functionality.
- **Detailed Modals:** Click on any issue card to view full details (author, status, priority, and description) in a sleek modal.
- **Responsive Design:** Fully optimized for mobile, tablet, and desktop views using Tailwind CSS.
- **Visual Indicators:** Category-based dynamic styling (Green borders for Open, Purple for Closed).

## 🛠 Tech Stack
- **HTML5:** Semantic structure.
- **Tailwind CSS (v4):** Modern utility-first styling.
- **DaisyUI (v5):** Interactive UI components.
- **Vanilla JavaScript (ES6+):** DOM manipulation and asynchronous API calls.
- **Vercel:** Deployment and hosting.

## 📚 Core JavaScript Concepts Used
This project implements several ES6+ features and fundamental JS concepts:

### 1️⃣ var vs let vs const
- **var:** Function-scoped, allows redeclaration and reassignment.
- **let:** Block-scoped, allows reassignment but no redeclaration.
- **const:** Block-scoped, no reassignment allowed, must be initialized at declaration.

### 2️⃣ Spread Operator (...)
- Used to expand, copy, or merge arrays and objects efficiently.

### 3️⃣ Array Methods: map(), filter(), forEach()
- **map():** Transforms an array and returns a new array.
- **filter():** Returns a new array containing only elements that meet a specific condition.
- **forEach():** Iterates through an array for side effects (like DOM updates), does not return anything.

### 4️⃣ Arrow Functions
- Provides a shorter syntax for writing functions and does not have its own `this` context.

### 5️⃣ Template Literals
- Using backticks (`) and `${}` for easy variable interpolation and multiline strings.
